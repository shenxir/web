# 三、系统设计

## 3.1 系统总体设计

### 3.1.1 系统架构设计

本系统采用经典的**前后端分离**架构，基于 B/S（Browser/Server）模式设计，分为三个主要层次：

```
┌─────────────────────────────────────────────────────────────┐
│                       表示层（Frontend）                      │
│  Vue 3 + Element Plus + Pinia + Vue Router                  │
│  运行于浏览器，负责用户界面展示和交互                           │
└─────────────────────────────────────────────────────────────┘
                              │ HTTP/RESTful API
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       业务逻辑层（Backend）                   │
│  Spring Boot 3.2 + MyBatis                                  │
│  Controller → Service → Mapper 三层架构                      │
│  处理业务逻辑、数据验证、权限控制                               │
└─────────────────────────────────────────────────────────────┘
                              │ JDBC
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       数据持久层（Database）                  │
│  MySQL 8.0 + Druid 连接池                                    │
│  存储系统所有业务数据                                         │
└─────────────────────────────────────────────────────────────┘
```

### 3.1.2 技术选型

| 层次 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| **前端框架** | Vue.js | 3.5.34 | 渐进式框架，易于上手，生态完善 |
| **UI 组件库** | Element Plus | 2.14.1 | 丰富的组件，适合后台管理系统 |
| **状态管理** | Pinia | 3.0.4 | Vue 3 官方推荐，轻量高效 |
| **路由管理** | Vue Router | 4.6.4 | 官方路由，支持路由守卫 |
| **构建工具** | Vite | 8.0.12 | 快速冷启动，热更新性能优异 |
| **后端框架** | Spring Boot | 3.2.5 | Java 生态主流，开箱即用 |
| **ORM 框架** | MyBatis | 3.0.3 | 灵活的 SQL 映射，适合复杂查询 |
| **数据库** | MySQL | 8.0 | 开源免费，性能稳定，社区活跃 |
| **连接池** | Druid | 1.2.21 | 阿里开源，监控功能强大 |

### 3.1.3 系统部署架构

```
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│    用户浏览器     │────▶│   Nginx 反向代理  │────▶│  Spring Boot     │

└──────────────────┘     └──────────────────┘     └────────┬─────────┘
                                                          │
```

**生产环境部署说明：**
- 前端打包后由 Nginx 静态托管

- 支持 Docker 容器化部署

## 3.2 功能设计

### 3.2.1 系统角色划分

系统定义两类用户角色，权限控制通过路由守卫和后端接口双重验证：

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| **普通用户（user）** | 学生/教职工 | 注册登录、发布物品、申请认领、查看个人信息 |
| **管理员（admin）** | 系统管理员 | 拥有普通用户所有权限 + 审核认领、管理物品、发布公告、用户管理 |

### 3.2.2 功能模块划分

系统划分为 **6 个核心功能模块**：

```
校园失物招领系统
├── 1. 用户认证模块
│   ├── 用户注册
│   ├── 用户登录
│   └── 密码修改
│
├── 2. 物品管理模块
│   ├── 发布失物/招领信息
│   ├── 物品列表浏览
│   ├── 物品详情查看
│   ├── 物品搜索筛选
│   └── 我的物品管理
│
├── 3. 认领管理模块
│   ├── 申请认领
│   ├── 我的认领记录
│   └── 认领审核（管理员）
│
├── 4. 分类地点模块
│   ├── 物品分类管理
│   └── 地点信息管理
│
├── 5. 公告管理模块
│   ├── 公告发布

└── 6. 数据统计模块
    ├── 物品统计
    ├── 认领统计
    └── 仪表盘展示
```

### 3.2.3 核心业务流程

#### （1）物品发布流程

```
┌─────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 用户登录 │────▶│ 填写物品信息 │────▶│ 上传物品图片 │────▶│ 提交发布    │
└─────────┘     └─────────────┘     └─────────────┘     └──────┬──────┘
                                                              │
                         ┌────────────────────────────────────┘
                         ▼
              ┌─────────────────────┐
              │   系统校验           │
              │  · 重复发布检查      │
              │  · 字段完整性校验    │
              │  · 自动生成过期日期  │
              └──────────┬──────────┘
                         │
              ┌──────────┴──────────┐
              │                     │
              ▼                     ▼
      ┌───────────────┐    ┌───────────────┐
      │   发布成功     │    │   发布失败     │
      │  状态:pending  │    │  返回错误信息  │
      └───────────────┘    └───────────────┘
```

**业务规则：**
- 同一用户 24 小时内不能发布相同标题的物品
- 物品发布后自动设置 7 天过期时间
- 支持多图上传（最多 5 张）
- 物品状态默认为 `pending`（待认领）

#### （2）物品认领流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 浏览物品列表 │────▶│ 查看物品详情 │────▶│ 申请认领     │
└──────────────┘     └──────────────┘     └──────┬───────┘
                                                 │
                                                 ▼
                                       ┌──────────────┐
                                       │ 填写认领理由  │
                                       └──────┬───────┘
                                              │
                                              ▼
                                       ┌──────────────┐
                                       │  系统校验     │
                                       │ · 是否已认领  │
                                       │ · 是否为本人  │
                                       │ · 物品状态    │
                                       └──────┬───────┘
                                              │
                               ┌──────────────┴──────────────┐
                               │                             │
                               ▼                             ▼
                     ┌─────────────────┐          ┌─────────────────┐
                     │   校验通过       │          │   校验失败       │
                     │  创建认领记录    │          │  返回错误信息    │
                     └────────┬────────┘          └─────────────────┘
                              │
                              ▼
                     ┌─────────────────┐
                     │ 等待管理员审核   │
                     └────────┬────────┘
                              │
               ┌──────────────┴──────────────┐
               │                             │
               ▼                             ▼
       ┌───────────────┐           ┌───────────────┐
       │   审核通过     │           │   审核拒绝     │
       │ 物品状态→     │           │  返回拒绝原因  │
       │   claimed     │           └───────────────┘
       └───────────────┘
```

**业务规则：**
- 同一用户对同一物品不能重复提交待审核的认领
- 认领申请需要管理员审核后才能生效
- 审核通过后，物品状态自动变为 `claimed`
- 认领人和物品发布者不能是同一人

#### （3）管理员审核流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 登录管理后台 │────▶│ 查看待审核   │────▶│ 查看认领详情 │
│              │     │   认领列表   │     │              │
└──────────────┘     └──────────────┘     └──────┬───────┘
                                                 │
                              ┌──────────────────┴──────────────────┐
                              │                                     │
                              ▼                                     ▼
                    ┌─────────────────┐                   ┌─────────────────┐
                    │    通过认领      │                   │    拒绝认领      │
                    │  · 添加审核备注  │                   │  · 填写拒绝原因  │
                    └────────┬────────┘                   └────────┬────────┘
                             │                                    │
                             ▼                                    ▼
                    ┌─────────────────┐                   ┌─────────────────┐
                    │ 更新认领状态     │                   │ 更新认领状态     │
                    │ → approved      │                   │ → rejected      │
                    │ 更新物品状态     │                   └─────────────────┘
                    │ → claimed       │
                    └─────────────────┘
```

### 3.3.3 数据库物理设计

#### 存储引擎
- 所有表使用 **InnoDB** 引擎，支持事务和外键约束

#### 字符集
- 统一使用 **utf8mb4** 字符集，支持中文和 emoji 字符

#### 外键约束关系

| 父表 | 子表 | 外键字段 | 级联操作 |
|------|------|----------|----------|
| sys_user | lost_found_item | user_id | RESTRICT, CASCADE |
| sys_user | item_claim | claimer_id | RESTRICT, CASCADE |
| sys_user | item_claim | reviewed_by | SET NULL, CASCADE |
| sys_user | sys_announcement | publish_by | SET NULL, CASCADE |
| sys_category | lost_found_item | category_id | RESTRICT, RESTRICT |
| sys_location | lost_found_item | location_id | RESTRICT, RESTRICT |
| lost_found_item | item_image | item_id | CASCADE, CASCADE |
| lost_found_item | item_claim | item_id | RESTRICT, CASCADE |

#### 软删除策略
- 用户表、物品表、公告表使用 `is_deleted` 字段实现软删除
- 图片表使用物理删除（级联删除）
- 查询时需过滤 `is_deleted = 0` 的记录

## 3.5 安全设计

### 3.5.1 身份认证

- 登录成功后生成 Token，存储在前端 localStorage
- 请求时通过 `Authorization: Bearer {token}` 头携带
- Token 过期或无效时，自动跳转登录页

### 3.5.2 权限控制

- **前端路由守卫**：根据路由 meta 字段（requiresAuth、requiresAdmin）控制页面访问
- **后端接口校验**：管理员接口校验用户角色

### 3.5.3 数据安全

- 密码传输：当前版本为明文存储（建议生产环境使用 BCrypt 加密）
- SQL 注入防护：使用 MyBatis 参数化查询
- XSS 防护：前端使用 Vue 的模板转义
- CSRF 防护：基于 Token 的认证天然防御

### 3.5.4 文件上传安全

- 限制文件大小：单文件最大 10MB
- 限制文件类型：图片格式（jpg、png、gif 等）
- 文件命名：时间戳 + UUID 防止重名

## 3.6 性能设计

### 3.6.1 数据库优化

- 合理使用索引，避免全表查询
- 使用 Druid 连接池管理数据库连接
- 大查询使用分页处理

### 3.6.2 前端优化

- 路由懒加载，减少首屏加载时间
- 图片懒加载，优化列表页性能
- 使用 Pinia 缓存热点数据（分类、地点）

### 3.6.3 接口优化

- 列表接口支持分页查询
- 使用批量查询减少 N+1 问题
- 合理使用缓存策略
